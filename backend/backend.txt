Lab instruction for Software Engineering students:
1. The primary focus of this lab is on: Front-end implementation (Back end if they have Started), Proper usage of GitHub for version control and collaboration.
2. Each student must clearly demonstrate her/his individual contribution through GitHub commits, branches, and pull requests. No marks will be awarded for untraceable contributions.
3. All projects must be maintained on GitHub. Create one common repository per group.
4. Repository naming format: Group No._Project Name
5. Repository must include: README.md, Proper folder structure, SRS Document
6. Each student must:
○ Create branches and commit,
○ Raise Pull Requests,
○ Merge branches properly
7. Commits must not be bulk uploads. Each commit must represent a logical change. Commit message format: <type>: short description
Examples:
feat: added login page UI
fix: corrected form validation logic
8. README.md must include:
○ Project title
○ Problem statement
○ Technology stack
○ Contribution table
